记录搭建博客的过程

2022/8/31 杂谈

本着好记性不如烂笔头的原则，加上马上读研究生了希望记录一下自己的学习过程，昨天花了半天时间使用Hexo搭建了属于自己的博客，也在这个过程中第一次接触到前端的东西。本篇就先记录一下如何使用Hexo框架搭建自己的博客。

前置准备

Hexo可以说是非常好用又简洁高效的博客框架了，安装也是非常方便的。但是在安装之前我们需要保证电脑里有这两个程序：

  1. Node.js 官网

  2. Git 官网

这两个都是非常常见的软件了，这里不展开介绍，如果没有安装的可以点击进入官网选择自己的电脑的版本进行安装，网上也有非常多的安装教程可以自己搜一下。

使用Hexo搭建博客

安装好Node.js和Git之后，我们就可以通过命令行愉快的进行各种安装啦！

安装Git后在桌面或者文件夹里邮件会多出两个选项，一个是Git GUI Here，另一个是Git Bash Here，前者是打开git的图形界面，后者是打开git命令行，我们通常直接使用命令行也就是Git Bash进行操作。那我们现在开始。

安装Hexo

在合适的位置（以后这个位置就作为博客的本地文件夹啦）新建一个文件夹，在文件夹中打开Git Bash，当然其他命令行也是可以的比如PowerShell，但是注意一般是要管理员权限的哦，输入以下命令：

#全局安装Hexo脚手架
npm install -g hexo-cli

安装完后可以输入

#查看hexo版本
hexo -v

查看是否安装成功，出现版本号就表示没问题啦！

第一次使用要初始化一下，可能会比较慢

hexo init

完成之后就搭建好了，分别输入下面的指令在本地进行预览

#Generate编译成静态文件
hexo g
#部署到本地
hexo s

成功后出现下面的提示，打开这个网址就可以看到博客的预览啦，里面有篇自带的Hello World

在博客中写文章

Hexo是支持Markdown语法的，也就是说，我们只需要在本地把文章按格式写在md文件里，Hexo就可以自动解析并生成设定好的好看的样式，在Hexo里写文章主要有两种方式。

第一种：使用命令创建文件
#生成一篇叫hello的文章
hexo new post "hello"

这个指令会在source/_posts/目录中生成一个”hello.md”文件，我们打开这个文件进行就可以了。当然我们也可以自己创建这个文件，也就是第二种方法
